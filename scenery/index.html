Scenery for Nagini

This page is the browser test suite for Nagini. The programmatic tests start on load and take a few minutes. run_tests.py drives this page headless and reads the hidden #test-results-json element once the run completes.

Test suite (this page)

  • Programmatic tests: automated checks for every class of the API, results in the table below.
  • Interactive tests: manual input(), matplotlib and Brython turtle demos.

Demo pages

  • examples/: showcase of the Nagini API with editable, runnable snippets.
  • executions/: one runnable example per execution flow, with the matching integration code and flow diagram.
  • lycee/: French high-school maths algorithms as runnable programs.
  • dataeng/: data engineering snippets (Python, SQL, streams); includes api-live/, FastAPI served in the browser.
  • arcade/: data engineer challenges and a sprint mode.
  • atelier/: theme workshop for daisyUI and CodeMirror.
  • atelier_rg2a/: RGAA contrast workshop on the same stack.

Legacy and local Pyodide pages

  • legacy/full_local.html: full local Pyodide (Capacitor/iOS offline mode); requires the untracked pyodide-local/ folder at the repo root.
  • legacy/full_local_needed_for_app.html: minimal local bundle (about 18 MB, SymPy and Pydantic only); requires the untracked pyodide-local-needed-for-app/ folder.
  • legacy/v0.0.26.html: esm.sh CDN import check pinned on v0.0.26, with the ?bundle flag.

Documentation

  • docs/execution-flows.md: step-by-step ASCII diagrams of every execution flow.
  • docs/architecture.md: system map, from host page to worker to build chain.
  • README.md: project overview, quick start and pinned CDN URLs.
